Problem: naming submissions, have name hierarchy? which?

  • flat name space without policy will lead to clashes over time
  • submission names are direcory names which are inflexible in CVS (no renaming/moving of dirs). If we decide on a hierarchy now, we haveto stick with it.

Hierarchy/policy options:

  • encode hierarachy in subdirectory or long dir name? (/a/b vs /a-b/)
  • what to use as classifier? (name, author, volume, year, ...?)

Examples:

  • name only: /Algebra
  • subdirectory per year / name only: /2003/Algebra
  • subdirectory per "volume": /1/Algebra
  • subdirectory per author/ name: /Klein/Algebra
  • name + author + year: /Algebra-GK04
Decision:
  • have flat namespace in CVS (we don't know yet what hierarchy will be good in 2 years time, and we don't want deep hierarchy without content)
  • submission-year based CVS structure might confuse people (might get mixed up with archive release years)
  • have chronological order on web page
  • have policy of reasonable, not too general names (not Algebra, but Ring)
  • in case of conflict (new submitter wants an existing name):
    • try to resolve by suggesting another reasonable name
    • if that fails append author/year or similar
